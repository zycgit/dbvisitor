# Method Annotations

By annotating Mapper interface methods, you **separate SQL from calling logic** without writing JDBC boilerplate code.

```java title='Define a Mapper interface'
@SimpleMapper
public interface UserMapper {
    @Query("select * from users where id > ?")
    List<User> listUsers(long searchId);
}
```

```java title='Use the Mapper'
Session session = config.newSession(dataSource);
UserMapper mapper = session.createMapper(UserMapper.class);
List<User> result = mapper.listUsers(2);
```

:::tip[Tip]
How you obtain a Session depends on your project architecture. See **[Framework Integration](../../yourproject/buildtools#integration)** for details.
:::

## Annotation List

| Annotation | Purpose | Example Scenario |
|------|------|--------|
| [@Query](./query) | Execute a query and return a result set | SELECT queries, pagination |
| [@Insert](./insert) | Execute an INSERT statement | Insert records, retrieve auto-generated keys |
| [@Update](./update) | Execute an UPDATE statement | Update records |
| [@Delete](./delete) | Execute a DELETE statement | Delete records |
| [@Execute](./execute) | Execute any statement (DML/DDL) | Create tables, batch operations, multiple result sets |
| [@Call](./call) | Call a stored procedure | Stored procedure/function calls |
| [@Segment](./segment) | Define a reusable SQL fragment | Referenced via `@{macro,...}` rules |

## Common Notes

- All annotations that accept SQL (`value` attribute) support **string arrays**; array elements are joined with spaces for multi-line readability
- SQL can use [Rules](../../rules/about) for dynamic capabilities (conditional concatenation, IN queries, etc.)
- The `statementType` attribute options are `Statement`, `Prepared` (default), and `Callable`
