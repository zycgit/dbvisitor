# Mapping Result Sets

Use either `<resultMap>` or `<entity>` to map query results:
- [&lt;resultMap&gt;](./result_map): map SQL result sets.
- [&lt;entity&gt;](./entity_map): map a concrete table.

:::tip[How to choose between resultMap and entity?]
- For the same table with different query shapes, `entity` is good when columns all come from one table; `resultMap` when columns span multiple tables.
- Using the [Fluent API](../lambda/about) requires `entity` or [Object Mapping](../mapping/about) annotations.
- If the SELECT uses aliases, functions, CASE, or columns from multiple tables, `resultMap` is safer.
:::

## resultMap tag

```xml title='Define and use resultMap'
<resultMap id="user_resultMap" type="com.example.dto.UserBean">
    <result column="id" property="id"/>
    <result column="name" property="name"/>
    <result column="age" property="age"/>
    <result column="create_time" property="createTime"/>
</resultMap>

<select id="queryById" resultMap="user_resultMap">
    select
        id, name, age, create_time
    from
        users
    where
        id = #{id}
</select>
```

### Attributes

**resultMap tag**

| Property                  | Description                                                                                                                                          |
|---------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|
| id                        | <TagGray/> Identifier; if empty, `type` is used.                                                                                                    |
| type                      | <TagRed/> Fully qualified Java type to map to.                                                                                                      |
| caseInsensitive           | <TagGray/> Case-insensitive mapping for column/property names. Default `true`.                                                                      |
| autoMapping               | <TagGray/> Whether to perform [auto-mapping](./automapping). Default `true`.                                                                        |
| mapUnderscoreToCamelCase  | <TagGray/> With auto-mapping, convert camelCase property names to snake_case columns (e.g., `createTime` → `create_time`). Default `false`.          |

**result tag**

| Property     | Description                                                                                                                                          |
|--------------|------------------------------------------------------------------------------------------------------------------------------------------------------|
| column       | <TagRed/> Column name in the result set.                                                                                                             |
| property     | <TagRed/> Property name in the target type.                                                                                                          |
| javaType     | <TagGray/> Usually inferred; specify when the property is abstract or an interface.                                                                  |
| jdbcType     | <TagGray/> Overrides the default [Java↔JDBC mapping](../../types/java-jdbc).                                                                         |
| typeHandler  | <TagGray/> Override the [type handler](../../types/about) to use.                                                                                    |
