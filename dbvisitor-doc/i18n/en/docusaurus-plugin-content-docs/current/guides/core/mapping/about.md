# Object Mapping

dbVisitor only does **Object Mapping**, not relational mapping (one-to-one, one-to-many, many-to-many, etc.). Java objects map directly to database tables/views, and database dialect differences can be abstracted away with the [Fluent API](../../api/lambda_api).

```java title='Quick Example'
@Table("users")
public class User {
    @Column(name = "id", primary = true, keyType = KeyType.Auto)
    private Long id;
    @Column("name")
    private String name;
}
```
