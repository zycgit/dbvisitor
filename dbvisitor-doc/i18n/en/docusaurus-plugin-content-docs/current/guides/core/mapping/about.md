dbVisitor strips out relational mapping and keeps only object mapping, simplifying database operations via the [Fluent API](../../api/lambda_api).

:::tip[dbVisitor object mapping highlights]
- Gentle learning curve without heavy concepts.
- SQL dialects hide database differences when using the [Fluent API](../../api/lambda_api).
- No relational mapping support (no 1-1, 1-n, n-1, n-n).
:::

## Guide

Mapping
- [Annotations](./basic): use @Table and @Column to map types to tables or views.
- [Mapper File](../file/entity_map): map via the &lt;entity&gt; tag in mapper XML.
- [Key Generators](./keytype): generate keys (sequences, UUID, or fetch auto-increment after insert).
- [TypeHandler](./type): handle enums, JSON, and special JDBC types in column mapping.

Tips
- [Camel case](./camel_case): auto-convert camel case to snake case.
- [Name sensitivity](./delimited): handle case sensitivity and reserved-word identifiers.
- [Write Policy](./write_policy): decide whether a field participates in INSERT/UPDATE when using Fluent API.
- [Statement Templates](./template): customize SQL fragments, e.g., MySQL `point` handling.
