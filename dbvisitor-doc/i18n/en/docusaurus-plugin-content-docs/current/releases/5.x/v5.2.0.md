# v5.2.0 (2023-01-15)

```xml
<dependency>
    <groupId>net.hasor</groupId>
    <artifactId>dbvisitor</artifactId>
    <version>5.2.0</version>
</dependency>
```

## Major changes
- Added `entity` tag in mapper files with `id` and `mapping` subtags.
- Removed `id` subtag from `resultMap`.
- Project layout: `dbvisitor-core` renamed to `dbvisitor`.
- Added more properties for guice/hasor/spring/spring-boot integrations.

## Added
- Primary key generator feature; supports DB auto-increment and custom generators.
- `allowUpdateKey` and `allowReplaceRow` to guard dangerous operations.
- `@DefaultTable` annotation (package-level) to batch-apply `@Table`.

## Improved
- Logging can be configured to print SQL execution logs.
- `DalSession.createMapper` can be called without pre-building `dalRegistry`.
- `JdbcOperations` cleaned up (removed inconsistent/rare methods).
- Upgrade cobble to 4.5.1.
- Faker MySQL metadata query improved to support richer geometry types.
- Faker adds Oracle data generation support.
- Removed `TypeSupplier` logic.
- Trimmed extra metadata queries in dbvisitor-faker.
- Mapping module enhanced for richer mapping descriptions.

## Fixed
- `queryForIterator` malfunction.
- Case sensitivity for Map/DTO parameter shapes.
