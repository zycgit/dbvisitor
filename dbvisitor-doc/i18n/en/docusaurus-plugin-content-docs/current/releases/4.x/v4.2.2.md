# v4.2.2 (2021-01-20)

```xml
<dependency>
    <groupId>net.hasor</groupId>
    <artifactId>hasor-db</artifactId>
    <version>4.2.2</version>
</dependency>
```

## Added
- Added a MyBatis-like set of `TypeHandler`s with more complete type coverage.
- Full support for JDBC 4.2 data types.
- Full support for Java 8 time types.
- Single-table ORM capabilities.
- Stored procedure support.
- `JdbcTemplate` adds lambda capabilities, allowing code-generated queries similar to MyBatisPlus.
- `JdbcTemplate` supports multiple SQL statements with multiple result sets.
- `JdbcAccessor` adds `dsApply` to control how `Connection` is obtained from `DataSource`, enabling external transaction managers.
- Auto-detection now recognizes `impala`.
- `ColumnMapRowMapper`/`ColumnMapResultSetExtractor` add a flag to control `caseInsensitive` behavior.
- Added `presto` connection strings to `JdbcUtils`.

## Improved
- Increased test coverage to 82%.
- `BeanPropertyRowMapper` now handles case-insensitive mapping.
- `hasor-db` no longer ships direct MyBatis integration wrappers.
- Relocated the `@Transactional` annotation within the `hasor-db` package.
